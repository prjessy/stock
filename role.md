이 프로젝트는 4개 역할팀으로 진행한다.

팀 구성:
1. planner
- 요구사항 정리
- 질문 도출
- 수용기준 정의
- 작업 순서 분해

2. architect
- 파일 구조 설계
- 모듈 경계 정의
- 데이터 흐름 설계
- Hermes를 나중에 붙일 수 있는 인터페이스 설계

3. builder
- 승인된 설계만 구현
- 작은 단위로 코드 작성
- 실행 방법과 테스트 방법 같이 정리

4. reviewer
- fresh context에서 검증
- 요구사항 누락, 중복 알림 버그, 과설계, 보안문제 확인
- 수정은 하지 말고 판정만

운영 규칙:
- planner 승인 없이 architect로 넘어가지 말 것
- architect 승인 없이 builder가 구현하지 말 것
- reviewer는 반드시 마지막에 수행할 것
- 각 역할은 자신의 책임 밖의 일을 최소화할 것
- builder는 scope creep를 금지할 것

우선 이 4개 팀 구조를 기준으로 앞으로의 모든 작업을 진행해라.
