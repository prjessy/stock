# 🔐 보안 수칙 — .env 는 절대 git 에 넣지 않는다

> **왜 중요한가:** `.env` 에는 KIS 앱키/시크릿, 카카오 키, Anthropic API 키 등이 들어 있다.
> git(특히 공개 GitHub `prjessy/stock`)에 한 번이라도 올라가면 **해커의 표적**이 되고,
> 커밋 히스토리에 남아 나중에 지워도 과거 기록에서 키가 그대로 노출된다.
> **키는 오직 VPS 의 `.env` 파일에만 존재해야 한다.**

---

## 핵심 규칙

1. **`.env` 는 커밋 금지.** 소스 배포는 항상 `.env` 를 **뺀** 채로 한다.
2. 키 원본(`key.txt`, `key2.txt` 등 `key*.txt`)도 커밋 금지.
3. 키는 **VPS 의 `~/stock-watchdog/.env` 에만** 둔다. 권한은 `chmod 600`(소유자만 읽기).
4. 새 PC/서버에 세팅할 때만 `.env.example` 을 복사해 값을 채운다 — `.env.example` 은
   **빈 템플릿**이라 추적해도 안전(시크릿 없음).

## 어떻게 보장되는가 (`.gitignore`)

```gitignore
.env          # 환경/시크릿
key*.txt      # KIS 앱키/시크릿/계좌 원본
```

→ git 이 **추적 자체를 안 함**. `git add .` 를 해도 `.env` 는 스테이징되지 않는다.

## 배포 시 .env 가 건드려지지 않는 이유

VPS 코드 갱신은 `git pull` 인데, **`git pull` 은 git 이 추적하는 파일만 갱신**한다.
`.env` 는 추적 대상이 아니므로 **VPS 의 .env 는 pull/배포로 절대 덮어써지지 않는다.**
소스만 안전하게 교체되고 키 파일은 그대로 유지된다.

```bash
# VPS 코드 갱신 (키 파일 영향 없음)
cd ~/stock-watchdog && git pull && bash deploy/deploy.sh
```

## 검증 결과 (2026-06-21 감사)

| 점검 | 결과 |
|---|---|
| `.env` 현재 추적 여부 | ✅ 추적 안 됨 |
| `.env` 과거 히스토리 노출 | ✅ 단 한 번도 커밋된 적 없음 |
| `key.txt` / `key2.txt` 추적 | ✅ 둘 다 추적 안 됨 |
| 추적 파일 내 KIS 키 패턴(`PS...`) 노출 | ✅ 없음 |

## 다시 검증하는 명령 (의심될 때 그대로 실행)

```bash
# 1) 현재 .env 추적되는가? (출력 없으면 안전)
git ls-files | grep -E "(^|/)\.env$"

# 2) 과거 히스토리에 .env 가 한 번이라도 있었나? (출력 없으면 안전)
git log --all --full-history --oneline -- .env

# 3) 커밋 전 스테이징에 .env 가 끼었는지 (출력 없으면 안전)
git diff --cached --name-only | grep -iE "\.env$"
```

## 만약 실수로 커밋/푸시했다면 (사고 대응)

1. **즉시 모든 키를 폐기·재발급한다** — KIS, 카카오, Anthropic. (히스토리에서 지워도
   이미 노출된 키는 살아있다고 가정)
2. 히스토리에서 제거: `git filter-repo` 또는 BFG 로 `.env` 를 전 히스토리에서 삭제 후
   강제 푸시.
3. `.gitignore` 에 등록돼 있는지 재확인.

> **순서가 중요:** 히스토리 청소보다 **키 재발급이 먼저**다. 청소는 시간이 걸리지만
> 그 사이에도 노출된 키는 악용될 수 있다.
