# 사용자 액션 가이드 — 아이폰 로그인 확인 & 구글 동의화면 게시

> 이 두 가지는 **코드/서버로 해결 안 되고, 사람이 직접** 해야 합니다.
> **둘 다 "오너(사장님) 본인"이 합니다.** 지인(상대방)이 하는 게 아닙니다.

## 누가 무엇을 하나 (한눈에)

| 작업 | 하는 사람 | 필요한 때 |
|---|---|---|
| 📱 아이폰 로그인 확인 | **오너 본인** | 로그인 버그 수정 검증(권장) |
| 🔓 구글 동의화면 게시 | **오너 본인** (Google Cloud) | 지인도 웹에 로그인할 경우 |
| 구글 계정으로 웹 로그인 | 지인(상대방) | 게시된 후, 본인 매매일지 쓸 때 |
| 텔레그램 알람 받기 | 지인(상대방) | 봇 `/start` (동의화면과 무관·이미 됨) |

---

## 📱 1. 아이폰 실기기 로그인 확인 (오너)

**왜**: 아이폰 사파리 전용 로그인 버그(`STATE MISMATCH`)를 고쳐 배포했는데, 실기기 검증은 사람이 해야 확정됩니다.

**방법**
1. 아이폰 **사파리**로 `https://jessystock.com` 접속
2. 📔 **매매일지** 탭 → **구글로 로그인**
3. 구글 계정 선택 → 동의 → **매매일지 화면이 뜨면 성공** ✅

**안 되면**: "state 불일치" 등 에러 메시지를 캡처해 공유 → 로그 대조해 추가 수정.

## 🔓 2. 구글 동의화면 '프로덕션' 게시 (오너)

**왜**: OAuth 동의화면이 **'테스트'** 상태면 **등록된 테스터(오너)만** 로그인됩니다.
지인들도 **각자 구글 계정으로 웹사이트 매매일지**를 쓰려면 **'프로덕션(게시됨)'**이어야 합니다.

> ⚠️ **텔레그램 알람과는 무관**합니다(알람은 봇으로 처리·이미 작동). 
> 이 게시는 **"지인도 웹 매매일지에 로그인"** 할 경우에만 필요합니다. 오너 혼자 쓰면 안 해도 됨.

**방법 (게시)**
1. [console.cloud.google.com](https://console.cloud.google.com) → **오너 구글 계정** 로그인
2. 상단 **프로젝트 선택** → 구글 로그인 키(클라이언트 ID)를 만든 그 프로젝트
3. 좌측 ☰ → **API 및 서비스 → OAuth 동의 화면** (새 UI: **Google 인증 플랫폼 → 대상(Audience)**)
4. 게시 상태가 **'테스트 중(Testing)'** → **[앱 게시 / PUBLISH APP]** 클릭 → **'프로덕션(In production)'**으로 전환
5. 끝. **이메일·프로필 스코프만 써서 구글 심사/검토 불필요 → 즉시 적용** ✅

**대안 (게시 없이, 소수일 때)**: OAuth 동의 화면 → **테스트 사용자(Test users) → 사용자 추가** → 지인 **구글 이메일** 입력(최대 100명). 게시 안 해도 그 사람들만 로그인 가능.

**참고**
- "확인되지 않은 앱" 경고 → **고급 → 계속(안전)** 누르면 로그인됨(기본 스코프라 안전).
- User type이 **External**이어야 외부 지인 가능(Internal=조직 내부 전용).

**확인**: 지인이 본인 구글 계정으로 jessystock.com 매매일지에 로그인되면 성공. (자세히: `docs/GOOGLE_LOGIN_SETUP.md`)

## 지인(상대방) 입장에서 할 일 (참고)

- **알람만 받을 경우**: 봇에 `/start` 한 번 → 오너에게 본인 ID 전달(또는 오너가 승인). 끝. (동의화면 게시 불필요)
- **웹 매매일지도 쓸 경우**: 위 #2 게시가 된 뒤 → jessystock.com 에서 **본인 구글 계정으로 로그인** → 각자 분리된 매매일지 사용.
