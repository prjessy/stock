# 🤖 자동매매봇 — 향후 가능 기능 (보류)

> **상태: 보류(향후 검토).** 대시보드 탭에서는 제거했습니다. 핵심(대시보드·미국 시장 동향·찰떡 써머리·더듬이·알람)을 먼저 안정화한 뒤, 필요해지면 다시 꺼내 구현할 수 있습니다.

## 왜 보류했나
- 실거래 주문은 **사고 위험·책임**이 크고, 핵심 가치(판단 보조·요약·알림)와 별개로 무겁습니다.
- 먼저 "판단 보조" 기능을 충분히 검증한 뒤, 그 위에 선택적으로 얹는 게 안전합니다.

## 만약 나중에 한다면 — 구상
조건 충족 시 자동 매수/매도 + 손절·익절 + 분할매매 + 알림 자동화. **모의 → 반자동 → 실거래** 순으로 단계적 활성화. (판단 보조이며 투자 책임은 사용자 본인)

### 기능 후보
| | 기능 | 내용 |
|---|---|---|
| ① | 종목 스캔 | 거래량 상위·급등락·조건식 필터 |
| ② | 전략 엔진 | 이동평균·RSI·볼린저 (교체 가능 인터페이스) |
| ③ | 주문 엔진 | 시장가·지정가·분할 매수/매도·체결확인 |
| ③-1 | **스케줄링 분할 매도 ⭐** | N주를 시점·가격대로 나눠 자동 분할 매도(장중 감시→단계별 집행). 증권앱이 못하는 핵심 기능 |
| ④ | 리스크 관리 | 손절·익절·트레일링스탑·예약 손절 스케줄·1회/일일 한도·동시보유 제한 |
| ④-1 | 예약 손절 스케줄 | 지정 시점/가격에 손절 자동 예약·집행 |
| ⑤ | 포트폴리오 | 보유·평균단가·평가손익·현금잔고 |
| ⑥ | 로그/리포트 | 주문·에러·전략 로그 + 성과 요약 |
| ⑦ | 알림 | 체결·손절익절·에러·장시작종료 (텔레그램) |
| ⑧ | 백테스트 | 승률·손익비·MDD·누적수익률, 전략 비교 |
| ⑨ | 운영 안정성 | API 재시도·예외 처리·안전 종료·이상 시 주문 중단 |

### 설계 (교체 가능 구조)
`config · broker adapter(한투/키움/토스 교체) · strategy(인터페이스) · risk manager · trader · notifier · logger · backtest`
→ 처음엔 mock broker로 검증, 실제 증권사 adapter만 추가.

### 원칙 (모델 A)
- 각 사용자가 **본인 키·계좌로 본인 환경**에서 실행 (운영자가 남의 키를 받지 않음).
- 모의/소액 검증 + 한도·킬스위치·로그 필수.
