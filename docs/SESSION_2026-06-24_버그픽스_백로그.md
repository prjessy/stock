## ✅ 이번에 고친 것 (코드 반영 · 배포 대상)

### #2 구글 로그인 "state 불일치(보안)" 실패
- **원인**: `api.py` 콜백이 OAuth `state` 를 **프로세스 메모리 셋(`_OAUTH_STATES`)** 에도 있어야 통과시켰음.
  서버 재시작·재배포·멀티워커 시 셋이 비어 정상 로그인도 'state 불일치'로 실패.
- **수정**: 메모리 셋 제거. **httponly `oauth_state` 쿠키 ↔ 쿼리 state 일치**(더블서밋)만으로 CSRF 방지 — 표준 방식, 재시작에 견고.
- 파일: `app/web/api.py` (`auth_google_login` / `auth_google_callback`, `_OAUTH_STATES` 삭제)

### 로그인 화면이 "갑자기" 전체화면으로 뜸 (mode 회귀)
- **원인**: 전체화면 게이트는 `mode === "full"` 일 때만. 서버 `_auth_mode()` 가 설정파일
  (`data/auth_config.json`, gitignore) 이 **없으면** `REQUIRE_LOGIN` env 폴백으로 `full` 을 반환.
  재배포로 `data/` 가 초기화되면 갑자기 전체 로그인 게이트가 떴음.
- **수정**: 폴백 기본값을 **항상 `journal`** 로. `full` 은 ⚙️설정에서 명시 토글했을 때만.
  → **지금처럼 "매매일지 누를 때만" 로그인** 동작 보장. (라이브 현재 mode=journal 확인됨)
- 파일: `app/web/api.py` (`_auth_mode`)

### #10 화살표 색 → 한국식 (적색=상승, 청색=하락)
- `--up: #2ec26b(초록) → #f6465d(빨강)`, `--down: #ff5d6c(빨강) → #4c8dff(파랑)`.
- 가격 외(非)용도 분리 위해 `--good`(초록)·`--bad`(빨강) 변수 추가 후 핀 고정:
  - AI 판단 **위험=빨강·적정=초록**(반전 방지), 위험 경고문, 캘린더 **일요일=빨강**.
  - 호가창: 매도호가=빨강·매수호가=파랑 (안내문 "빨강=매도/파랑=매수"와 일치하도록 매핑/문구 정렬).
  - 상태 메시지: 성공(✅)=초록(`--good`)·실패(❌)=빨강(`--bad`)로 정리.
- 파일: `app/web/static/index.html`
- **남은 follow-up(소)**: `innerHTML` 안에 인라인으로 박힌 "조회 실패/스캔 실패" 등 **약 15곳**이
  `color:var(--down)` 라 현재 **파란색**으로 표시됨(가격 라벨 '▼ 하락'과 같은 변수를 공유해 일괄치환 위험).
  필요 시 `--bad` 로 개별 이관(저위험·반복작업).

### #4 대시보드 수급 "데이터 안 나옴"

  → 코드 버그 아님. 일시적 KIS 빈응답/로딩 타이밍으로 추정.

- 파일: `app/web/static/index.html`

### #9 자동매매 패널 "종가 가격 안 나옴(0원)"

  자동매매 탭으로 바로 들어오면 시세가 아직 없어 `(?? 0)` 로 **0원** 표시.
