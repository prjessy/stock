## 1. VOC — 사용자가 원하는 것 / 불만

### 핵심 요구
- **손절·익절 자동화**: 직장인이 수시 모니터 불가 → 급락 대응. 가장 중요한 안전장치.
- **서버 기반 자동감시주문**: 키움 "자동감시주문"(서버 스탑로스)이 표준 니즈 — HTS 안 켜도 24시간 조건 실행. (← 우리 앱이 이미 하는 것)
- **분할매수/매수한도**: 리스크 관리용 암묵적 요구.

### 증권사 평가
| 증권사 | 자동매매 | 특징 |
|---|---|---|
| 키움 | ★★★★★ | 자동감시주문·조건검색·알람 업계 최고. UI는 복잡 |
| 한국투자 | ★★★ | OpenAPI 제공, 자동매매 기능은 미흡, 낮은 신용금리 |
| 대신 | ★★ | 정보 부족 |

공통 불만: **속도·안정성·조건검색 정확도**.

### "AI가 자동으로 수익" 서비스 현실 반응 (로보어드바이저, 2026)
- 자문·일임 규모 3,483억(3배↑). 성과 양호 사례: 콴텍 국내주식형 +35.26%(코스피 -9.68% 대비).
- **그러나**: ① "AI가 계속 수익" 기대 → 실망. 단기 급등 맞추기 불가. ② 같은 회사 내 **알고리즘 간 수익 편차 수십 %p**. ③ 실제 가치 = **하락장 방어(리스크 관리 도구)**, 상승장 언더퍼폼 경향.
- 사용자 후기 요지: "단기 급등 맞히는 도구가 아니라 **변동성 줄이는 리스크 관리 장치**."

### 사용자가 중시하는 안전장치
1. 손절 자동화(필수) 2. 일일 매매 한도·최대손실액 3. 분할매수 수량 제한 4. 오류로 인한 다중주문 차단.

## 2. 급등락 사전 탐지 — 기술 실현성

### 신호별 예측력
| 신호 | 예측력 | 비고 |
|---|---|---|
| 거래량 급증 | 중상 | 주가 선행, 폭증 시 상승 가능성↑ |
| 변동성 돌파(Volatility Breakout) | 중상 | 심층신경망 결합 시 개선 |
| 공시/뉴스 | 높음 | 신뢰 높으나 이미 공개정보(선행성 제한) |
| 호가 불균형 | 중하 | 단독 예측 곤란(가격발견 과정) |
| 모멘텀/RSI | 중하 | 후행지표, 지연 |
| 외인·기관 수급 | 낮음~중 | **단기 X, 지속 추세 ○** |
| 프로그램매매 | 낮음 | 파생 변화로 급변 |

핵심: "**지속적 수급동향**을 꾸준히 체크"가 단기 호가보다 중요.

### 구조적 한계 (해결 불가에 가까움)
- **Look-ahead bias(선행편향)**: 백테스트가 과도 낙관 → 실전 실패. 신호는 시점 t에 가용한 정보만 써야.
- **과적합**: 신호↑·모델 복잡↑ = 과적합 악화. train/val/test 분리해도 근본 해결 X.
- **시장 비정상성**: 과거 패턴 ≠ 미래. "과거 1년 모델 ≠ 현재 3개월."

### 현실적 정확도
- 단일 신호 40~60%, **3신호 이상 결합 60~70%(조기경보 수준)**, 공시·뉴스 수동검증 추가 시 70~80%.
- **단정적 예측("10:30에 A 급등") 불가** → **확률적 조기경보**가 한계. **자동매매보다 "조기 알림 + 수동 판단"이 현실적.**

출처: koreascience.kr 변동성돌파 심층신경망, arxiv 1912.07165(intraday jumps), velog Look-ahead Bias, arxiv 2512.23847(LLM forecast lookahead), brunch 과적합.

## 3. 자율 자동매매 — 법적·기술 제약 (한국)

### 법
| 구분 | 정의 | 개인 가능 |
|---|---|---|
| 프로그램매매 | 동시 **5억↑** 주문 | ✗ (기관/외국인, 예탁원 승인) |
| 알고리즘 트레이딩 | 5억 **미만** 자동 | ✓ 가능(거래소 제약 미미) |

- 개인이 OpenAPI로 자동매매: **법적 OK**(알고리즘 트레이딩).
- 프로그램 **판매** 시: 단순 도구+기본설정 제공=안전 / 개인 자산·목표 반영 **맞춤 권고 = 투자자문업 규제**.

### 기술 — KIS API
- **초당 20건 호출 제한**(엄격). 50종목+ 동시감시 시 정체.
- 다중종목 실시간 스캘핑/고빈도 **불가**(수초 래그). **단일·소수 종목 + 수초 틱은 OK** ← 우리 구조(소수 종목·15초 틱)는 문제없음.

출처: braindetox.kr 토스 OpenAPI, tgparkk.github.io robotrader(20건 제한), hky035.github.io throttling.

## 결론 (우리 앱 방향)
- "AI 완전자율 보장 수익"은 비현실 → **규율 있는 조기경보 + 리스크 관리 자동매매**로 포지셔닝.
- 급등락은 **확률적 조기경보(60~70%)**로만. look-ahead/과적합/비정상성 주의.
- 우리 구조(소수 종목·15초 틱)는 KIS 제약 안전. 다종목 초단타는 불가.
- 필수 안전장치: 손절·총한도·일일손실 정지(킬스위치)·모의 선검증.
