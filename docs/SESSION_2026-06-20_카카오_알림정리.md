## 📍 현재 알림 구조 (오늘 파악한 현황)

| 알림 | 트리거 | 발송처 | 브라우저 필요? |
|---|---|---|---|
| 🎯 목표가 도달 | 설정한 종목별 목표가 도달 | 텔레그램 + 카카오 | **필요** (클라이언트 1초 폴링) |
| 🚨 더듬이 AI 신호 | 본장(평일 09:00~15:30) **10분마다 자동** 감시 → 매수/매도 신호 | **텔레그램만** | 불필요 (서버 24h) |
| 📰 아침 브리핑 | 매일 07:00 | 텔레그램 | 불필요 (서버) |
| 🔔 ±% 급등락 뱃지 | 카드 ±임계값 도달 | **메시지 없음(화면 뱃지만)** | — |

- VPS 설정: `DEUDEUMI_INTERVAL_MIN=10`, `ANTHROPIC_API_KEY` 설정됨 → 더듬이 자동감시 **동작 중**
- 급등락 ±%는 더듬이 트리거 조건(|change_pct|≥3%)으로 **간접 연결**(AI 분석 후 신호 시 발송), 단독 ±% 알림은 없음

## ⚠️ 핵심 한계 (왜 알림이 약한가)

1. **목표가 알림이 클라이언트(브라우저)** — 대시보드를 열어둬야만 동작. 서버 상시 감시 아님.
2. **카카오가 서버 감시에 연결 안 됨** — 더듬이 AI 신호는 텔레그램만. 그래서 지인(카카오)은 목표가만 받음.
3. **카카오 단일 토큰** — 토큰 1개라 사장님·지인 동시 수신 불가(현재 지인 1명). 둘 다 카톡 받으려면 멀티유저 개발 필요.

## 🔜 다음 세션에 할 일

**목표: hermes(24h 서버 텔레그램)처럼, 알림을 전부 "서버 24시간 감시"로 통일.**
(이미 있는 `app/analysis/deudeumi_scheduler.py` 패턴 재사용)

```
서버 스케줄러
   ├─ 24h 워치리스트 감시 (목표가 · 급등락 · AI신호)
   ├─ 텔레그램 → hermes        (사장님)
   └─ 카카오   → kakao_notify  (지인)
```

### 결정해야 할 것
1. **어떤 알림을 서버 24h로?** (복수 선택)
   - ⓐ 목표가 도달 (종목별 지정가)
   - ⓑ 급등락 ±% 단독 알림 (예: ±3% 넘으면 즉시, AI 없이)
   - ⓒ 더듬이 AI 신호를 **카카오에도** 추가 (지금은 텔레그램만)
2. **목표가 저장 위치** — 서버 감시하려면 목표가를 **서버(.env/DB)에 저장**하도록 변경 필요 (현재 브라우저 localStorage에만 있음)

### 작업량 가늠
- **ⓑ + ⓒ (가장 가성비)**: 목표가 서버이전 없이 바로 24h 양쪽 발송 — 약 반나절
- **ⓐ 목표가 서버 감시**: 저장 구조 변경(설정 UI→서버) 추가 — 별도 반나절
- **카카오 멀티유저**(사장님도 카톡): 사용자별 토큰 저장 — 별도 반나절

### 추천 시작점
**ⓑ 급등락 ±% + ⓒ 더듬이→카카오** 부터. (브라우저 의존 없이 바로 24h, 양쪽 채널)
