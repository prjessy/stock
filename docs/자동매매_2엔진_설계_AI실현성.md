# 자동매매 2엔진 설계 + AI 실현가능 범위 (2026-06)

내일 다시 검토용. "입금 + 조건 + 시동 → AI가 알아서 매매"를 향한 **두 엔진** 설계와, **AI가 현실적으로 어디까지 가능한지** 판정.

---

## AI 실현가능 범위 — 정직한 판정
- ✅ **가능**: 2엔진 분리, 대가리(고점) 감지(발목 대칭), 조건 자동매매(있음), AI 적정/위험/중립 판단(있음), 자율성 3모드 옵션, **반복 밴드/그리드 매매(매도가 기준 N회)**, 조기경보 알림, 안전장치(한도/일손실 킬스위치), 모의 선검증.
- ⚠️ **조건부**: 급등락 "사전" 탐지는 **확정 예측 불가 → 확률적 조기경보(~60-70%, 거짓신호)**. 수년 데이터는 백테스트/보정엔 유효하나 look-ahead·과적합 주의.
- ❌ **비현실/주의**: "AI가 알아서 무조건 수익" **보장 불가**(로보어드바이저도 리스크관리 도구, 알고리즘 편차 큼). 다종목 실시간 초단타는 **KIS 초당 20건 제한**으로 불가(우리는 소수종목·15초틱이라 OK). 유료판매+맞춤권고 = 투자자문업 규제.

→ 포지셔닝: **"규율 있는 조기경보 + 리스크관리 자동매매"** (보장수익 X). 모의 검증 후 소액 실전.

## 두 엔진 구조
```
[엔진1] 판단기 (AI 두뇌)          [엔진2] 실행기 (손)
"바닥/대가리/급등락?"   ──계약──▶  "받은 계획대로 주문"
→ TradePlan(매수/매도, 진입가/구간,            → KIS 주문 + 포지션 관리
   수량/금액, 익절%, 손절%, 근거)    ◀─체결/포지션 피드백─
```
- **엔진1**: bottom_detect(발목)·top_detect(대가리, 신설)·feed·deudeumi_ai·ai_advisor 조립 → TradePlan 생성. confidence 상한 0.7.
- **엔진2**: autotrade_watch 리팩터(판단X, plan 소비) + kis_order. risk_guard 게이트.
- **통신**: contracts.py(TradePlan/Signal dataclass) + plan_store.py(data/trade_plans.jsonl 버스). 직접호출 대신 신호 저장소.

## 자율성 3모드 (사용자 옵션)
- **A**: 내 워치리스트 안에서 AI가 진입가·수량·익절손절·타이밍 결정.
- **B**: 종목 발굴까지 AI (완전자동).
- **C**: AI 추천 + 사람 버튼 승인 (기본·가장 보수적).

## 반복 매매 (오늘 구현분 + 확장)
- **1회 자동**: 사용자 지정가(+AI 적정/위험/중립 보조) → 자동 매수, 조건 매도.
- **N회 반복(완전자동 그리드)**: 1차 기준 = **①종가기준(전일대비 등락률, 자동)** 또는 **②사용자 지정가(AI 보조)**. 매도 후 2차~는 **직전 매도가 기준 ±증감율%**로 매수/매도 N회.
- 상태: data/band_state.json (또는 메모리, 거래일 리셋).

## 안전장치 (risk_guard, 실전 필수)
킬스위치 / 총 투자한도 / 1종목·1회 한도 / **일일 손실한도→자동정지** / 모의 선검증 / qty 하드캡(기존 유지).

## 신규 파일(계획)
contracts.py · plan_store.py · risk_guard.py · top_detect.py(대가리) · engine1.py(판단 조립) · surge_detect.py(급등락 조기경보).

- Phase 2: top_detect + engine1 + /api/top-scan
