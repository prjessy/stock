## 1. 종목 추가/관리 UI ⭐ (가장 큼: 백엔드+프론트)

현재 워치리스트는 `.env`(`kr_symbols`/`us_symbols`)에 **3개 고정**. 화면에서 추가/삭제 불가.

### 백엔드
- [ ] **동적 워치리스트 저장소**: `data/watchlist.json`(gitignore)이 .env를 오버라이드.
      없으면 .env 기본값으로 시드. (라이브 VPS에만 존재, 공개 레포엔 안 올림)
- [ ] `SourceRegistry`/`RealtimePoller`가 워치리스트를 **동적으로 재읽기**(현재 init 시 1회 캐시 → 변경 반영 안 됨).
- [ ] API: `GET /api/watchlist`, `POST /api/watchlist`(추가), `DELETE /api/watchlist/{symbol}`.
- [ ] **심볼 검증/이름 조회**: 추가 시 6자리=국내(KIS `inquire-price`로 유효성+종목명), 그 외=미국(yfinance)로 확인 후 등록. 실패 시 거부.
- [ ] 중복/개수 상한(예: 20개) 가드.

### 프론트
- [ ] ⚙️ 설정창(또는 전용 영역)에 **종목 검색 + 추가**, 현재 목록 + **삭제(✕)** 버튼.
- [ ] 추가/삭제 즉시 카드/표 갱신.

### 주의
- 동시성: 폴러(1초)와 워치리스트 변경이 겹칠 수 있음 → 저장소 읽기/쓰기 락 또는 원자적 교체.
- KIS 토큰 1분 1회 제한: 신규 종목 검증도 기존 토큰 재사용(새 발급 X).

## 3. 목표가 알림 ⭐ (중간)

현재 알림 = ±% 등락(서버 `.env` 임계값 → 텔레그램) + 클라 표시 임계값(localStorage). 
**목표가 도달 알림** 추가: "삼성 80,000원 도달 시 알림".

- [ ] **저장**: 종목별 목표(가격/방향: 이상·이하). 서버측(텔레그램 발사 위해) — `data/targets.json` 또는 DB.
- [ ] **체크**: 폴러/스케줄러가 시세 갱신 시 목표 도달 판정(중복 발사 방지 = 도달 후 비활성/쿨다운).
- [ ] **알림 채널**: 기존 텔레그램(`app/core/alerts.py`) 재사용 + 대시보드 카드 강조 + (선택)브라우저 알림.
- [ ] API: `GET/POST/DELETE /api/targets`.
- [ ] 프론트: 종목 카드/설정창에서 목표가 입력·목록·삭제.

> ±% 알림과 목표가 알림은 별개 트랙(둘 다 텔레그램으로). 메시지에 어떤 트리거인지 표기.

## v2 — 예측/포착 엔진 "더듬이" 🐜 (비전)

매수/매도 타이밍을 잡는 3개의 "더듬이"(감지기). 출력은 **사이렌/팝업/텔레그램** 알림.

### 더듬이 1 — 피보나치 다단계 (규칙 기반, AI ❌)
- 최근 고저 구간의 피보나치 되돌림(23.6/38.2/50/61.8/78.6%)으로
  **1차·2차·3차 매수가**, **1차·2차·3차 매도가** 산출.
- 결정적·투명(수식). 차트에 레벨선 표시 + 도달 시 알림.

### 더듬이 2 — 변동장 시점 포착 (AI 판단)
- 요즘 변동성 장세를 **Claude가 분석**(추세/거래량/변동성/패턴)해 매수·매도 시점 포착 → **사이렌**.
- ⚠️ "심층 학습"=별도 모델 학습이 아니라 **Claude in-context 분석**(최근 데이터를 매 주기 입력해 판단). 자체 ML 모델 학습은 별도 대형 과제.

### 더듬이 3 — 종목 추이 학습 적용 (AI 판단)
- 삼전·하닉 **최근 그래프 추이를 집중 분석**해 해당 종목 특성에 맞춘 신호 → **사이렌**.
- 마찬가지로 Claude in-context(최근 N일 봉/지표 요약 입력).

### 알림 방법 / 옵션
- **방법**: 사이렌(소리) · 팝업 메시지 · 텔레그램.
- **옵션**: 단순 알람(소리/뱃지만) vs 메시지 전달(맥락·근거 포함 텍스트/텔레그램).
- 텔레그램 활성화 선행: `.env` TELEGRAM_BOT_TOKEN/CHAT_ID + 알림 데몬 기동.

## v3? — 자동매매 (옵션, 신중)

KIS OpenAPI 주문 API로 본인 계좌 자동매매. **단계적·안전 우선**.
- KIS 주문: 매수 `TTTC0802U` / 매도 `TTTC0801U`, 잔고 `TTTC8434R` 등.
- **경로**: ①모의투자 계좌(가짜돈) 검증 → ②반자동(신호=사이렌+알림, 매매는 사용자 클릭) → ③소액 실거래 자동 + 안전장치(1회 최대금액·일손실 한도·종목당 한도·킬스위치).
- 기술 난이도 중, 진짜 리스크는 **버그=손실·전략 정확도·규제**(남의 돈 운용은 라이선스 필요, 개인 계좌는 본인 책임).
- 먼저 **반자동**까지만 만들어 테스트 권장.
