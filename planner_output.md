# 차트 탐지견 — Planner 산출물 (v1.2)

> 앱 이름: **차트 탐지견** 🐕 (급등락 "냄새"를 미리 맡는 조기경보 컨셉). 원칙: **선택과 집중 — 종합선물세트형 앱 지양.**

> 상태: 요구사항·범위 정리 + 개념설계 초안. **코드 미작성.** architect 상세설계 진입 전 일부 Open Question 미해결.

## 1. 목표
로컬 PC·VPS 어디서나(코드는 GitHub 동기화) 실행 가능한 **알림 전용** 주식 보조 시스템.
- 국내 2종목(+레버리지 2종) + 나스닥 선물의 가격 증감 알림
- 종목별 재무분석 + 마케팅 자료 취합
- 아침 미국 선물 동향 브리핑
- 대시보드로 매일 확인
- 전송·명령은 **Hermes 게이트웨이 경유 Telegram**(양방향)
- **자동매매 없음**
- **주 목적(확정)**: **매매 판단 보조** — 방향 예측/매매 지시가 아니라, 남보다 빠르고 맥락 있는 정보로 사용자의 판단을 돕는 의사결정 보조 도구. 최종 판단·책임은 사용자. (15분 지연=판단 보조용, 실행 타이밍용 아님)

## 2. 감시 대상 (선택과 집중)
- 국내 2종목: 삼성전자(005930), SK하이닉스(000660)
- 국내 레버리지 2종: KODEX 삼성전자레버리지(0193W0), KODEX SK하이닉스레버리지(0193T0) — 카드에 교육필수·비용·음의복리 경고 표시
- 미국: **나스닥 선물(NQ=F)만** — 아침 프리장 방향성 확인용
- (제거) 마이크론(MU)·필라델피아 반도체지수(^SOX): 종합선물세트 지양 위해 제외

## 3. 확정 요구사항
- R-1 감시 대상은 위 2종목 + 3지표
- R-2 기준가: 각 종목 전일 종가
- R-3 임계값: 설정 리스트, 기본 `[+3%, -3%]`, 코드 수정 없이 확장(±6/±9 등)
- R-4 중복 방지 키: (거래일, 종목, 임계값) — 당일 1회
- R-5 아침 미국 선물 브리핑: 지정 시각 1회
- R-6 알림/명령 채널: Hermes 게이트웨이 경유 Telegram (outbound 알림 + inbound 명령)
- R-7 환경 비종속: 로컬·VPS 공통, 설정은 env/`.env`, secrets 하드코딩 금지, GitHub 동기화(`.env`·상태DB 커밋 제외)
- R-8 발송 이력 영속화(재시작 후 중복 방지)
- R-9 자동매매 없음
- R-10 최소 의존성·유지보수 우선
- R-11 종목별 재무분석(주기적)
- R-12 종목별 마케팅 자료 취합
- R-13 대시보드로 매일 확인
- R-14 가격 증감 시 Hermes→Telegram 알림
- R-15 모든 작업 지시를 Hermes Telegram 봇으로 수행(명령 라우터)

## 4. 기술 스택 (확정)
- 언어: **Python**
- 저장소: **SQLite** (저장소 인터페이스 추상화 → 추후 MySQL/Postgres 전환 가능)
- 사유: 국내 시세/재무 데이터 생태계(FinanceDataReader, pykrx, OpenDartReader, yfinance)가 Python 중심, pandas 기반 분석 용이, 최소 의존성.

## 5. Acceptance Criteria
- AC-1 종목별 임계값 첫 도달 시 1폴링 주기 내 Hermes→Telegram 알림 도착
- AC-2 같은 (거래일, 종목, 임계값) 재발송 없음; 다른 종목/임계값은 독립
- AC-3 임계값을 설정만 바꿔도 코드 변경 없이 동작
- AC-4 재시작 후에도 당일 보낸 조건 미발송(이력 영속)
- AC-5 거래일 변경 시 기준가·이력 초기화
- AC-6 시세/전송 일시 실패 시 프로세스 유지, 다음 주기 정상
- AC-7 가격 알림 메시지에 (종목명, 코드, 현재가, 전일종가, 등락률, 시각) 포함
- AC-8 아침 브리핑이 지정 시각에 1회 발송
- AC-9 전송 한쪽 실패가 다른 동작을 막지 않음(부분 실패 격리)
- AC-10 동일 코드가 로컬·VPS에서 env만 바꿔 동작
- AC-11 토큰/키가 `.env`에서 로드, 소스·git 미노출
- AC-12 명령(예 /price /brief /status)이 Telegram에서 수신·처리·응답됨

## 6. 개념 설계 (모듈 경계 — 코드 없음)
- Hermes Gateway: outbound 알림/리포트 → Telegram, inbound 명령 → Command Router
- Notifier Interface: Core는 "보낸다"만 인지, 구현체=Hermes (교체 가능)
- Command Router: 텍스트 명령 파싱 → Core 서비스 호출 → 응답 회신
- Core Services: Threshold Engine, Dedupe/State Store, Scheduler(폴링/아침브리핑/일일리프레시), Financial Analysis, Marketing Aggregator, Briefing Generator
- Data Source 어댑터: 국내 시세 / 미국 시세·지수·선물 / 재무 / 마케팅 (출처 교체 가능)
- Dashboard: 읽기 뷰
- Config/.env: secrets

## 7. 확정된 결정 (Decisions log)
- **Hermes**: VPS에 이미 셋업된 **AI 에이전트**. Telegram 전송 테스트 완료, Claude API 결제 대기(곧 해결). → 우리 시스템은 Hermes에 데이터를 넘기고 Hermes가 Telegram 중계.
- **앱↔Hermes 연결**: 둘 다 같은 VPS → **내부 localhost HTTP**. 외부 노출 없음(보안).
- **대시보드 접속(외부)**: 지금 `VPS IP:포트` → 추후 **Cloudflare 도메인**(무료 SSL/CDN/IP 숨김).
- **대시보드 기술**: **FastAPI**(JSON API) + 가벼운 정적 프론트 + **결과 캐싱**(무거운 분석은 미리 계산해 SQLite 저장).
- **규모**: 상용 아님, **지인 공유용 소규모** → 무거운 인프라 불필요, **가벼운 접근제한**(비번/허용 사용자)만.
- **재무분석 범위**: **기본 지표**(PER, PBR, 시가총액, 매출·영업이익, 배당) **+ 차트분석**.
- **차트분석 범위**: **가격 차트(캔들/라인)+거래량 + 기본 보조지표(MA, RSI, MACD)**. 차트는 데이터만 백엔드에서 주고 프론트에서 렌더.
- **마케팅 자료 취합 = 4종**: ① 뉴스 기사 ② 공시/IR 자료 ③ 애널리스트 의견·목표가 ④ SNS/커뮤니티 언급. (SNS·목표가는 안정적 무료 출처가 제한적 → "되는 출처부터" 단계적 연결)
- **아침 미국 브리핑**: 매일 **07:00 KST**, 내용 = 마이크론(MU)·필라델피아 반도체지수(SOX)·나스닥 선물(NQ) 동향(전일 종가·야간 선물 등락).

- **텔레그램 명령(3종)**: ① 현재가 조회 ② 브리핑 온디맨드 ③ 종목 분석 요청. (Hermes가 AI라 자연어 입력 → Hermes가 의도 해석 후 우리 앱 호출) 시스템 상태/설정 변경은 후속.
- **지연 고지(R-17)**: 무료 소스는 약 15분 지연 → 대시보드 배지 + **알림/브리핑 메시지 푸터에도 "약 15분 지연" 고지** 포함(KIS 실시간 전환 시 제거). 알림이 지연 데이터 기반임을 항상 명시.
- **무인 자동 운영(R-19, 토스 차별화 핵심)**: 한번 설정하면 끝 — 상시 데몬이 **매 거래일 전일종가 자동 리베이스 + ±% 조건 자동 재무장**. 사용자가 매일 알림을 다시 설정할 필요 없음("set & forget"). 이미 **AC-5**(거래일 변경 시 기준가·이력 초기화) + **중복키(거래일,종목,임계값)** 설계에 반영됨. → 컨셉 "탐지견(깔아두면 알아서 냄새 맡음)"과 일치.
- **맥락 있는 알림(R-18, 토스 차별화)**: ±% 알림 메시지에 단순 등락률만이 아니라 **① 나스닥 선물 프리장 방향 ② 지연 고지**를 함께 담는다. **뉴스/공시 맥락**은 builder-5(마케팅 취합) 연동 후 단계적 추가. 레버리지 경고는 **대시보드 카드에만** 표시(알림 메시지엔 미포함). 증감율은 옵션(기본 ±3%, `.env THRESHOLDS`로 다단계 확장).

## 9. 추가 기능 — 급등락 조기경보 (R-16, 토스 차별화 핵심)
- **목표**: 단순 사후 ±% 알림(토스류)을 넘어, 급등락 **직전 신호**를 탐지해 **미리·단계별**로 알린다.
- **솔직한 범위**: "예측"이 아니라 **확률적 조기경보**. 과거 급등락 직전 자주 나타난 조건을 탐지. **오경보 불가피** → 백테스트로 적중률·오경보율 측정·보정.
- **차별점**: ① 조기성 ② 다중신호(거래량·변동성·모멘텀/RSI·갭·이격) ③ **교차시장 선행지표**(미국 SOX·MU·나스닥선물 야간흐름 → 국내 반도체 다음날) ④ 단계별(관심/주의/경고) ⑤ 근거+뉴스/공시 맥락.
- **단계(작업)**: ew-1 수년치 데이터 수집 → ew-2 신호 엔진 → ew-3 백테스트·보정 → ew-4 단계별 경보 알림.
- **선행**: 설계(태스크 #10) 승인 후 ew-1~4 순차 진행. 알림은 기존 Hermes→Telegram 경로 재사용.
