# 분할 매매 자동 스케줄링 — 가이드

증권앱이 못 하는 **"여러 번 나눠서·정해진 시점/가격에 자동 매수·매도"** 를 KIS OpenAPI로 구현.
모델 A(각자 본인 키·계좌로 본인 환경에서 실행) 기준. **판단 보조/실행 도구**이며 책임은 사용자.

---

## 1. 필요한 일 (선행조건)

### 1-1. 계좌·키
- [ ] **KIS 계좌** + **앱키/시크릿** (주문 권한). KIS 개발자포털에서 발급.
- [ ] **계좌번호**: `CANO`(앞 8자리) + `ACNT_PRDT_CD`(상품코드, 보통 `01`)
- [ ] **실전 vs 모의** 선택
  - 모의: KIS 모의투자 신청 → 도메인 `openapivts...:29443`, tr_id `V`접두 (가짜돈, 안전)
  - 실전: 기존 도메인 `openapi...:9443`, tr_id `T`접두 (**진짜 돈**)
  - 테스트는 **모의 또는 실전 소액(1주)** 권장

### 1-2. 설정 (`.env`, git 금지)
```
KIS_APP_KEY=...
KIS_APP_SECRET=...
KIS_DOMAIN=https://openapi.koreainvestment.com:9443   # 모의면 vts 도메인
KIS_CANO=12345678
KIS_ACNT_PRDT_CD=01
KIS_ENV=real        # real | paper  (tr_id 접두 결정)
```

### 1-3. 실행 환경
- [ ] **24시간 켜진 서버** (스케줄링은 상시 가동 필요 — VPS/상시 PC/라즈베리파이)
- [ ] 텔레그램 봇 (각 분할 집행 알림/확인용) — 이미 hermes로 검증됨

### 1-4. 사전 합의 (안전)
- [ ] 1회 최대 수량/금액, 일일 한도
- [ ] **드라이런(모의 발주) 기본 ON**
- [ ] 킬스위치(즉시 전체 중지)
- [ ] 모든 주문 로그 보관

---

## 2. 구현할 내용

### 2-1. KIS 주문 모듈 (`app/trading/kis_order.py`)
- 토큰 재사용(기존 `KisPriceSource` 토큰 캐시 활용)
- **읽기전용(위험 0)**: 잔고조회 `TTTC8434R`, 주문가능 `inquire-psbl-order` ← **여기부터 시작**
- 매도 `TTTC0801U`(실전)/`VTTC0801U`(모의), 매수 `TTTC0802U`/`VTTC0802U`
- 정정·취소 `TTTC0803U`, 체결조회
- 모든 함수 실패 시 예외 격리 + 결과 반환

### 2-2. 분할 규칙 정의 (사용자 입력)
```
종목, 총수량, 매수/매도, 분할 방식:
  - 시간분할: [10:00 30주, 11:00 30주, 13:00 40주]
  - 가격분할: [80,000원 ↑ 시 30주, 82,000 ↑ 40주, ...]
  - 비율분할: 총 100주를 3회(30/30/40%)
```
→ JSON/DB로 저장 (`data/trade_plans.json`, gitignore)

### 2-3. 스케줄러 (`app/trading/scheduler.py`)
- APScheduler 또는 기존 cron 구조 재사용
- 매 주기: 활성 계획 점검 → 시점/가격 조건 충족 → 해당 분할분 주문
- **중복 집행 방지**(이미 집행한 분할은 완료 마킹)
- 장 시간/세션 체크(`app/core/market.py` 재사용)

### 2-4. 안전장치 (필수)
- **드라이런 모드**: 주문 직전까지 시뮬 → 실제 발주 X (기본값)
- 1회 수량/금액 캡, 일일 한도 초과 시 차단
- 킬스위치 플래그(파일/엔드포인트) → 즉시 전체 중지
- 체결 실패 재시도(백오프) + 실패 알림
- 모든 주문·결과 로그 (`data/trade_log.jsonl`)

### 2-5. 텔레그램 연동
- 각 분할 집행 시: "🚨 삼성 30주 매도 집행 (10:00, 81,200원)" 알림
- **반자동 모드**: 집행 전 "지금 30주 매도? [예/아니오]" 확인 받고 진행
- 킬스위치도 텔레그램 명령으로

### 2-6. 피드백 기록 (진화 씨앗)
- 모든 주문: 계획가 vs 체결가, 시점, 결과를 기록
- 나중에 "내 분할 전략 성적" 분석 → 더듬이/비서가 학습

---

## 3. 안전한 진행 순서 (단계)
1. **읽기전용**: 잔고·주문가능 조회만 (실거래 0, 키·계좌 연동 확인)
2. **드라이런**: 분할 계획 → 발주 직전까지 시뮬(로그만)
3. **실전 1주**: 텔레그램 확인 후 진짜 1주 (검증)
4. **반자동 분할**: 각 분할 집행 전 텔레그램 확인
5. **자동 분할**: 신뢰되면 안전장치 켠 채 자동 (소액→점진)

> 1~2단계는 위험 0이라 바로 시작 가능. 3단계부터 진짜 돈 → 한도·확인 필수.

---

## 4. 시작점
지금 막힌 것 = **계좌번호(`.env`에 `KIS_CANO`/`KIS_ACNT_PRDT_CD`)**.
넣으면 **1단계(읽기전용 잔고·주문가능 조회)** 부터 안전하게 착수.
